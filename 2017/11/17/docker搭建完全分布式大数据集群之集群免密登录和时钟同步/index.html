集群通信的一些准备工作

集群免密登录

集群免密登录是大数据集群工作的基础，其原理不是很复杂，简单来说就是私钥保留公钥分发用于登录验证，每台机都会存有自己的私钥以及整个集群所有机器的公钥。比较笨的方法是挨个机器的生成公钥和私钥，然后挨个的拷贝，这样能达到效果，但是太浪费时间而且没啥意思，我的做法是用shell一次性搞定。

由于一开始必然要涉及到跨集群登录，甚至是操作，因此还是需要引入一些辅助工具类，这里主要用的是expect，安装shell:

下面逐渐说明每个文件的内容。

hosts：

内容很简单，就是主机名加上对应的ip地址。

create_rsa.sh

作用是创建本机的秘钥文件，密码是空，不需要手动输入任何东西。

scpnopas.exp

主要的作用是使用expect，将hadoop1的一些后续工作需要的文件拷贝到指定机器的指定目录下。

remote_create_ssh.exp

脚本有两个作用，一是登录到目标机器，切换到root用户安装expect；二是执行先前拷过去的创建秘钥的脚本，创建本机的私钥和公钥。具体的一些操作在shell中均作了详细的说明。

chmod600.exp

将目标机器上认证文件赋予600的权限，这也是实现ssh免密登录的重要步骤。

create_ssh.sh

这是实现集群所有机器免密登录的核心脚本，具体的步骤思路是

  • 1 将必要的文件拷贝到集群其他机器上
  • 2 远程登录到集群中其他机器上，执行脚本。生成，各个集群自己的公钥和私钥
  • 3 将远程机器的所有公钥拷贝到hadoop1上，合并，然后分发到集群各个机器上
  • 4 远程登录每一台机器，将认证文件的权限赋为600

ntp同步

ntp同步的原理就是配置一个server，其他slave不断的跟这个server进行同步，server端可以是互联网的具体时间，也可以是本地集群某一台机器的时间。本例中将hadoop1配置为server端，集群其他机器都与hadoop1连接同步时间。

所有机器上安装ntp，并设置为开机启动

while read line
do
OLD_IFS=”$IFS”
IFS=”:”
array=($line)
IFS=”$OLD_IFS”
ip=${array[1]}
hostname=${array[0]}
./remote_install_ntp.exp $hostname

done < ./hosts

同步硬件时间

在 Linux 下系统时间在开机的时候会和硬件时间同步(synchronization),之后也就各自独立运行了，那么既然两个时钟独自运行,那么时间久了必然就会产生误差了,而 NTP 默认又只更新系统时间,因此我们需要设置硬件时钟进行同步调整
