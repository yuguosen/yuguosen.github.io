Docker搭建分布式大数据环境之CentOS的一些初始化设置

发表于 2017-10-09 | | 阅读次数:

前言

在Docker搭建分布式大数据环境之准备工作中我们成功将CentOS系统写入到移动硬盘中。进入之后机会发现在这个最小系统里面真的是什么都没有，ifconfig这些基本的命令也没有，不知道自己的ip，也ping不通外网，很蛋疼啊。接下来我们就来进行一些操作让我们的CentOs系统真正的飞起来。

网络设置

网络是必须的，一般的设置都是那种开机自动随机ip，这个病不符合我们的需求，因为后面的操作基本上都是使用xshell远程操作（我是这样的，因为在黑漆漆的CentOS里面搞事情确实不爽，有时候需要查一些资料什么的就非常的蛋疼），因此需要的是固定ip。

编辑CentOS的网络配置文件

cd  /etc/sysconfig/network-scripts

进入到目录后会发现一个ifcfg-eth0文件

vi ifcfg-eth0

将文件中如下几个字段改成如下的内容：

BOOTPROTO=static #dhcp改为static   
ONBOOT=yes #开机启用本配置  
IPADDR=192.168.1.62 #静态IP，根据自己需要设定  
GATEWAY=192.168.1.1 #默认网关  
NETMASK=255.255.255.0 #子网掩码  
DNS1=192.168.1.1 #DNS 配置 
DNS2=8.8.8.8

然后还需要修改/etc/sysconfig/network文件增加网关如下：

GATEWAY=192.168.1.1

重启网络服务：

systemctl restart network

完成后使用如下命令查看IP:

ip address

需要说明的是一般情况下我们使用的是ifconfig来查看ip，但是由于这些东都没有装所以使用原生的命令即可。这时候就可以试着ping自己的网关192.168.1.1，以及ping百度，如果能ping通就万事大吉，接下来就直接通过xshell连接就行。连上之后首先做的事就是要关闭防火墙，因为后面要干很多坏事：

systemctl stop firewalld.service #关闭防火墙
systemctl disable firewalld.service #禁止firewall开机启动

更新yum源

推荐使用阿里的yum源，具体步骤如下

安装wget

yum install -y wget

下载repo文件

wget http://mirrors.aliyun.com/repo/Centos-7.repo

备份并替换系统的repo文件

cp Centos-7.repo /etc/yum.repos.d/ 
cd /etc/yum.repos.d/ 
mv CentOS-Base.repo CentOS-Base.repo.bak 
mv Centos-7.repo CentOS-Base.repo

执行yum源更新命令

yum clean all 
yum makecache 
yum update

装一些基本软件

yum install -y vim 
yum install net-tools

ok,有网，一些基本的软件有了，基本的环境就差不多了，接下来就可以直接开始进入主题了

Docker搭建分布式大数据环境之准备工作

发表于 2017-10-09 | | 阅读次数:

前言

由于之前的博客比较混乱，在简书和CSDN上都写过，体验感觉都不怎么好（尤其是CSDN现在强制要求绑定手机号），索性自己开始使用Hexo+GitHub构建自己的博客。以前的部分文章在后续看能不能有时间搬过来。

系列文章说明

大数据现在已经成为互联网”三大俗”之一了，虽然感觉快要烂大街了，但是还是处在风口之上，很多IT狗（例如我）对其还是趋之若鹜。但说实在的大数据的门槛还是比Java的要高些，别的不说，首先一个完全分布式的大数据环境，就让不少人比较头疼。随便去某云上买个主机一年下来都要不少钱。而本系列文章的目的就是将我自身的解决方案分享出来，大家一起来愉快地搞学习。用到的主要Docker，这玩意儿从2013年蹦出来之后热度就一直蹭蹭的往上涨。但是关于Docker的详细信息以及高级技术，不在本系列讨论之类（其实是自己完全的小白，就知道怎么照葫芦画瓢，哈哈哈）。

正式开始

基础硬件环境准备

  • 笔记本一个。内存8G，这个要求在现在来看已经是很低了吧。
  • 移动硬盘一个。我自己是在京东上买的1T的西部数据，300多一点，具体大小视情况而定
  • Linux系统盘（U盘）一个。这玩意很简单，如果跟我一样上手的时候是个小白的话，我的建议是不要虚，撒开了搞，错了再格式化。而且手上有一两个系统盘，基本上是程序猿的标配了。

有了这些东西以后，我规划的未来是这样的：首先将一个Linux系统写入到你的移动硬盘上，再在这个Linux系统上安装Docker引擎，然后通过Docker虚拟出你想要的集群。是不是很简单？是不是很美好？来吧，一起开始填坑。

制作Linux系统盘

这一步网上的教程多的是，工具也是五花八门，我用得是UltraISO，优点是比较干净。然后Linux镜像的话我用的事CentOS7系列，原因的话，因为公司的云服务器是阿里云，它用的是CentOS，So就这样了。这里还要说一点的是这个系统是你最底层的Linux系统，相当于地基，建议使用CentOS7，比较新而且稳定。建议新手下一个minimal版的可以顺便练一下Linux的一些基本操作，我使用的CentOS7地址：http://isoredirect.centos.org/centos/7/isos/x86_64/CentOS-7-x86_64-Minimal-1708.iso
下载之后就直接使用UltraISO即可。

将CentOS7系统安装到移动硬盘

将制作好的Linux系统盘（U盘）和移动硬盘（如果是新的需要格式化一下，文件格式好像没什么特殊的要求）插到电脑上，重启电脑，然后按F12即可进入启动项,选择你的U盘启动（如果是金士顿的话就是那个以KingSton开头的U盘）。进入之后就可以看到CentOS的安装引导界面，选择第一个Install CentOS。进入后首先是选择语言，英语即可，第二步的设置就比较多一些,主要就是选择磁盘，将我们的CentOS写入到我们的移动硬盘上,点击select device即可看到如下界面:

选择自己的移动硬盘即可搞定。然后再是设置时区、root密码等基本设置。这些设置完成之后等待几分钟即可，会提示reboot。同样的，F12这次选择的是我们的移动硬盘启动，成功进入之后即表示整个CentOS系统已经装成功。
至此，我们已经有了一个初步的CentOS系统，这是我们往后工作的基础，当然啦我们选择的是最轻量级的系统，基本上是啥都没有。不过不用着急后面我们会慢慢一步步根据我们的具体需要装一些东西，逐步完善这个Linux系统。
